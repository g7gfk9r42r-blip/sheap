- Location: Berlin, Germany
- Tech-savviness: High

**Quote:** *"I want to eat well and save money, but I don't have time to hunt for deals across 5 different apps."*

## User Goals Summary

### Primary Goals
1. **Quick Daily Overview**: See what's planned for today at a glance
2. **Meal Inspiration**: Discover recipes based on current offers
3. **Habit Tracking**: Track hydration and daily mood/reflection
4. **Cost Savings**: Understand how much money is being saved

### Secondary Goals
1. **Weekly Planning**: Get inspiration for weekly meal planning
2. **Wellness Awareness**: Reflect on daily eating and wellness habits
3. **Recipe Discovery**: Find new recipes that match current offers

### Pain Points
1. **Information Overload**: Too many options can be paralyzing
2. **Time Pressure**: Need quick decisions, not endless browsing
3. **Context Switching**: Don't want to jump between multiple screens
4. **Motivation**: Need gentle nudges, not pushy notifications

## Context of Use

### Primary Contexts
1. **Morning Routine** (6-8am)
   - Quick check of today's plan
   - Set daily intention/mood
   - Review hydration goal
   - **Needs**: Fast, clear, non-intrusive

2. **Meal Planning Session** (Sunday afternoon)
   - Browse weekly highlights
   - Plan meals for week
   - Check current offers
   - **Needs**: Exploration mode, more information density

3. **Quick Check** (Throughout day)
   - Add hydration glass
   - Quick recipe reference
   - **Needs**: One-tap actions, minimal friction

### Device Context
- **Primary**: Mobile phone (iOS/Android)
- **Secondary**: Tablet (for meal planning sessions)
- **Constraints**: Thumb-friendly interactions, readable on small screens

---

## Key Insights

1. **HomeScreen is the "Command Center"**: Users expect to see everything they need for the day in one place
2. **Empty States Matter**: When no meal is planned, this is an opportunity, not a failure
3. **Progressive Disclosure**: Don't show everything at once - prioritize today's needs
4. **Gentle Guidance**: Users want help, not pressure
5. **Quick Actions**: Most interactions should be 1-2 taps maximum
